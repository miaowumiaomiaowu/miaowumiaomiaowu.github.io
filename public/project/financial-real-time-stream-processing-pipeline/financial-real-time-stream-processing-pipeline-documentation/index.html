🌊 金融实时流处理与AI预测系统

Financial Real-Time Stream Processing Pipeline

面向真实交易环境打造的企业级金融实时流处理与AI预测系统

🎯 项目概述

本项目是一个完整的金融科技解决方案，专为真实交易环境设计。系统通过微服务架构实现了数据采集、实时流处理、AI预测和可视化监控的完整闭环，拒绝模拟数据，坚持使用真实市场数据进行预测和分析。

🔥 核心特色

  • 🚀 真实数据驱动: 集成yfinance、akshare等真实API数据源
  • 🤖 企业级AI预测: 基于ml_training训练的真实模型，非随机模拟
  • ⚡ 高性能流处理: 毫秒级WebSocket实时数据推送
  • 🏗️ 微服务架构: 8个核心服务，支持独立扩展和部署
  • 📊 全栈可观测性: Prometheus + Grafana监控体系
  • 🎨 现代化界面: React + TypeScript前端仪表盘

🧩 服务组件详解

服务名称 端口 技术栈 职责
API Gateway 8000 FastAPI HTTP/WebSocket网关，业务路由聚合
AI Service 8002 FastAPI + ML 真实训练模型推理服务
Streaming Service 8004 aiohttp 实时流处理与消息推送
Data Collector 8003 FastAPI 实时/批量数据采集
Risk Management - Python 风控与审计模块
Frontend 3000 React + Vite 用户界面与数据可视化
Monitoring 3001 Grafana 系统监控与告警

✨ 核心功能

🔄 实时数据流处理

  • 统一消息格式: {type, data, timestamp, version} envelope协议
  • 多源数据融合: yfinance、akshare、自定义API
  • 毫秒级延迟: WebSocket实时推送
  • 容错机制: 自动故障转移和数据回退策略

🤖 AI预测引擎

  • 真实模型: 基于RESSET数据离线训练的生产级模型
  • 多策略预测: 技术分析、基本面分析、情感分析
  • 实时推理: 支持秒级预测响应
  • 模型版本管理: 支持A/B测试和模型热更新

📊 监控与可观测性

  • 系统健康: /api/v2/system/* 健康检查端点
  • 业务指标: 实时交易量、预测准确率、系统延迟
  • 告警机制: 基于Prometheus的智能告警
  • 性能分析: 详细的链路追踪和性能分析

🔐 企业级特性

  • 内部鉴权: JWT + 内部服务认证
  • 数据安全: 端到端加密和敏感数据脱敏
  • 高可用: 多实例部署和负载均衡
  • 扩展性: 容器化部署，支持K8s编排

🚀 快速开始

环境要求

  • Python 3.11+
  • Node.js 16+
  • Docker & Docker Compose
  • 8GB+ RAM推荐

一键启动

# 1. 克隆项目
git clone https://github.com/miaowumiaomiaowu/Financial-Real-Time-Stream-Processing-Pipeline.git
cd Financial-Real-Time-Stream-Processing-Pipeline

# 2. 环境配置
cp .env.template .env
# 编辑 .env 文件配置数据库和API密钥

# 3. 快速启动（推荐）
python quick_start.py --mode all

# 或者分步启动
python scripts/start_docker.py      # 启动基础设施
python scripts/start_backend.py     # 启动后端服务
python scripts/start_frontend.py    # 启动前端服务

Docker生产部署

# 生产环境部署
docker-compose -f docker-compose.production.yml up -d

# 国内用户加速
docker-compose -f docker-compose.cn-mirror.yml up -d

📡 API文档

REST API端点

系统监控

GET /api/v2/system/health      # 健康检查
GET /api/v2/system/status      # 系统状态
GET /api/v2/system/metrics     # 系统指标

AI预测服务

POST /api/v2/ai/predict        # 实时预测
GET  /api/v2/ai/models         # 模型列表
POST /api/v2/ai/train          # 模型训练

数据服务

GET  /api/v2/data/stocks       # 股票数据
GET  /api/v2/data/realtime     # 实时行情
POST /api/v2/data/collect      # 数据采集

WebSocket端点

实时数据流

// 连接实时数据流
const ws = new WebSocket('ws://localhost:8000/ws/realtime');

// 消息格式
{
  "type": "stock_price",
  "data": {
    "symbol": "AAPL",
    "price": 150.25,
    "change": 1.25,
    "volume": 1000000
  },
  "timestamp": "2024-01-01T12:00:00Z",
  "version": "1.0"
}

内部广播API

# 向所有WebSocket客户端广播消息
curl -X POST http://localhost:8000/api/internal/ws-broadcast \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <internal_token>" \
  -d '{"type": "alert", "data": {"message": "系统维护通知"}}'

📊 数据流架构

数据采集策略

外部API数据源 (优先级1)
├── yfinance API        # 美股数据
├── akshare API        # A股数据
└── 自定义API          # 其他金融数据

↓ 故障转移机制

历史训练数据 (优先级2)
└── RESSET数据集       # 仅当所有API失效时使用

数据处理管道

原始数据 → 清洗过滤 → 特征工程 → 模型预测 → 结果推送
    ↓         ↓         ↓         ↓         ↓
  验证格式   去噪异常   技术指标   AI推理   WebSocket

环境变量配置

# API密钥
ALPHA_VANTAGE_API_KEY=your_api_key
POLYGON_API_KEY=your_polygon_key

# 服务配置
API_SERVICE_PORT=8000
AI_SERVICE_PORT=8002
STREAMING_SERVICE_PORT=8004

# 安全配置
JWT_SECRET_KEY=your-secret-key
INTERNAL_API_TOKEN=your-internal-token

模型配置

# ml_training/config.py
MODEL_CONFIG = {
    "prediction_horizon": "1h",      # 预测时间窗口
    "feature_window": "24h",         # 特征时间窗口
    "retrain_interval": "7d",        # 重训练间隔
    "confidence_threshold": 0.75,     # 置信度阈值
}

🚨 故障排除

常见问题

1. WebSocket连接失败

# 检查服务状态
curl http://localhost:8004/health

# 查看服务日志
docker logs financial-streaming-service

2. AI预测服务异常

# 验证模型文件
ls -la ml_training/models/

# 重新训练模型
python ml_training/train_models.py

3. 数据采集中断

# 检查API配额
python tools/check_api_limits.py

# 切换数据源
export FALLBACK_DATA_SOURCE=true

性能优化建议

  • 数据库: 为频繁查询的字段添加索引
  • 缓存: 增加Redis内存分配
  • 并发: 调整uvicorn workers数量
  • 网络: 使用CDN加速静态资源
